# 分析调试环境

ida载入分析， 选择mips-big，

tip

qemu对mips的构架有四种，64/32 + big/small， 其中默认qemu-system-mips为32位大端， 64位标注64, 小段后缀l，

调试使用gdb，这里可以自己编译一个比较新版本的gdb并指定--target=mips选项，也可以直接安装gdb-multiarch, 在archcn源是有这个的，然后使用这个gdb运行并attach调试即可，

在run.sh文件里加入-s选项，表示给出一个gdbserver的端口， 配合gdb进行调试，

这时候我们使用gdb载入进去，设置构架和大小端， target指定附加即可。gdb中命令：

set architecture mips
set endian big 
target remote 0.0.0.0:1234
